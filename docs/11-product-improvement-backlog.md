# Product Improvement Backlog

> **Purpose:** Prioritize user-facing product improvements that increase value, retention, and differentiation in the current no-auth phase.

---

## P0 — Core Product Loop (Now)

| ID | Improvement | Why | Acceptance Criteria |
|---|---|---|---|
| P0-1 | Seed high-quality spots in Trieste + Milan | App is only useful if users find real spots immediately | At least 10 spots per city with valid coordinates, equipment, and descriptions |
| P0-2 | Spot-to-plan flow | Connect discovery to training outcome | From a selected spot, user can generate a plan using that spot's equipment |
| P0-3 | Spot filters that match real behavior | Faster discovery improves engagement | City + equipment filters are visible, fast, and persist for session |
| P0-4 | No-auth trust signals | Improve confidence without accounts | Show review count, average rating, last updated date, and seeded "verified" marker |

## P1 — Engagement & Retention (Near Term)

| ID | Improvement | Why | Acceptance Criteria |
|---|---|---|---|
| P1-1 | Plan history and quick regenerate UX | Encourage repeat usage in same session | User can revisit generated plans and regenerate with 1 click |
| P1-2 | Lightweight onboarding hints | Reduce first-session confusion | Add contextual hints for map filters and plan form |
| P1-3 | Analytics-driven funnel iteration | Prioritize by behavior, not intuition | Dashboard tracks visits -> spot views -> plan generations |
| P1-4 | Landing page CTA alignment | Increase conversion to core actions | Primary CTA leads to spots/plan, not auth screens |

## P2 — Differentiation Features (After Stabilization)

| ID | Improvement | Why | Acceptance Criteria |
|---|---|---|---|
| P2-1 | City leaderboard v1 | Reinforce hyper-local competitive angle | Leaderboard page with top users/activity per city |
| P2-2 | Streak mechanics (no-auth temporary) | Improve short-term retention | Session/local streak concept tested with basic UX |
| P2-3 | Reintroduce auth/profile intentionally | Enable persistence and gamification scale | Auth/profile returns with migration + updated UX flows |

---

## Product Rule

Before adding new features, ensure each release strengthens the core loop:
**Discover spot -> generate plan -> complete workout -> return.**
